# Que 画面生成プロンプト（image-2 用）

> デスクトップブラウザモック（MacBook）で統一。Home（`designs/home-desktop-final.png`）を基準とし、同じデザイン言語・情報密度・トーンを共有する。

---

## 共通プリアンブル（全画面で先頭に置く）

Que という日本の資格学習 Web サービスの画面モックアップ。デスクトップブラウザ（MacBook）で開いた画面の正面プロダクトショット。

サービス内容: IT パスポート・基本情報技術者・SPI・簿記2級の4資格に対応した、スマホ／タブレット／PC で使える資格学習サブスク（月額980円）。ターゲットは20代女性、キャリアアップ志向、スタバで分厚い参考書を広げて勉強するタイプ。分厚い参考書に挫折した層を拾う。

デザイン哲学は「Apple が資格学習サービスを作ったらこうなる」。Apple 純正アプリに全振り、引き算、コンテンツ主役、静けさと余白が主役。トーンは静かで上品、Apple が「Study, elevated.」のキャプションで出しそうな佇まい。教育サービス感・マーケ感・絵文字・マスコット・装飾グラデ・重いドロップシャドウは禁止。日本語 UI、柔らかなオフホワイト背景、MacBook のブラウザ枠込みで1枚出力。

左サイドバーのナビは Home 画面と共通（Today / コース一覧 / 復習リスト / AI Q&A / 音声解説 / ブックマーク / 学習履歴、下部にプレミアムカードとユーザープロフィール）。

---

## 1. コース一覧（Learn）

（共通プリアンブル）

この画面は「コース一覧」。参照する Apple 純正アプリは **Apple Books**。横スクロールシェルフと抽象的な表紙カードで、資格ごとのコースを棚のように並べる。

含まれる最低限の要素: 画面タイトル、4資格（IT パスポート / 基本情報技術者 / SPI / 簿記2級）ごとの横スクロールシェルフ、各コースの抽象アートカバー・タイトル・進捗バー、画面上部に「続けて学ぶ」のセクション。Home の抽象的な山・波のアートワークと同じ作法で表紙を描き、ストア感ではなく静かな学習棚の雰囲気にする。

---

## 2. 問題演習（Practice / Focus）

（共通プリアンブル）

この画面は「問題演習」画面で、1問に集中させるモード。参照する Apple 純正アプリは **Apple Journal**。余白を最大まで取り、問題文を中央に置き、ツールバーは最小限。

含まれる最低限の要素: 中央寄せの問題文、選択肢（3〜4つ、クリックで選択できるリスト）、画面上部に残り問題数と進捗バー、右上に小さなブックマークと問題番号、最下部に「次へ」ボタン。左サイドバーは Home と共通のまま残し、本文エリアだけ Journal 的に余白を大きく取る。情報密度は抑え、静けさを最優先。

---

## 3. 学習サマリー（Summary）

（共通プリアンブル）

この画面は「学習サマリー」画面。参照する Apple 純正アプリは **Apple Fitness**。3リングを画面の主役まで拡大し、過去実績を時系列で可視化する。

この画面は Home の右側にあった「学習サマリー」ウィジェットをそのまま全画面に拡張したもの。画面の主役は大きな3リング（問題数 / 正答率 / 学習時間）、週別の棒グラフ、期間タブ（週 / 月 / 年）、ストリーク（`flame.fill`）。弱点分野カードや目標設定の導線は副次的な補助要素として下段に控えめに置き、上段の静けさを邪魔しない。

---

## 4. AI Q&A

（共通プリアンブル）

この画面は「AI Q&A」画面。参照する Apple 純正アプリは **Apple Journal**。逆時系列の会話カードと、インラインのプロンプト提案で構成される。

含まれる最低限の要素: 画面下部に質問入力欄（プレースホルダ `何でも質問してください`）、入力欄の上に提案プロンプト（`HTTPとHTTPSの違いを教えて` `正規化の3段階を要約して` など）、画面中央に過去の Q&A が逆時系列で並ぶカード、各カードに参照元（関連ノートや章の名前を短く）、回答には根拠への遷移リンク。

---

## 5. 夜の音声解説クイズ（Now Playing）

（共通プリアンブル）

この画面は「夜の音声解説クイズ」の再生画面。参照する Apple 純正アプリは **Apple Music の Now Playing**。アートワークを主役に、波形と再生コントロールで構成する。

含まれる最低限の要素: 中央に大きな正方形アートワーク（夜空・月・静かな光の抽象画、Home で使った月イラストの世界観）、その上にタイトル（例: `今日のまとめ ／ データベースの正規化`）・サブタイトル（資格名）、波形シークバー、再生時間と残り時間、再生 / 前へ / 次へ の必要最小限のコントロールのみ。音楽アプリの装備（シャッフル・リピート・AirPlay・長大なトラックリスト）は入れず、学習音声を夜に一本聴く体験に絞る。暗めのトーンでも良いが、ダークモード全振りは避けトーンを上品に。

---

## 6. Playground（基本情報技術者のみ）

（共通プリアンブル）

この画面は「Playground」で、基本情報技術者の科目B（実技）向けのコード実行環境。参照する Apple 純正アプリは **Swift Playgrounds**。左エディタ / 右結果のミニマル配置。

この画面は開発 IDE ではなく、学習者が環境構築なしにコードを試せる優しい実験場。**Swift Playgrounds の柔らかい雰囲気を最優先**。画面上部に問題タイトルと設問文、中央を左右分割し左にシンプルなコードエディタ（軽い等幅フォント、控えめなシンタックス彩色）、右に実行結果パネル、右上に `実行` ボタン（pill 形 systemBlue）、下部にヒント、左サイドバーは Home と共通。Xcode・VS Code 的な機能密度は避け、コンソールやターミナル的な暗い背景は使わない。
